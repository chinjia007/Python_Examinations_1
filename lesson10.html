一、 知识点大回顾：我们学了什么？

知识讲解：

同学们，我们已经一起学习了9节课的Python知识啦！现在我们来一起回忆一下都学了哪些重要的内容，确保大家都牢牢记住了。

第1课：Python与IDLE初识
  • Python是什么？（一种高级编程语言）
  • IDLE是什么？（Python自带的开发和学习环境）
  • IDLE的两种模式：交互模式 (>>>) 和脚本模式。
  • 如何新建、保存 (.py 后缀)、打开和运行Python文件。
第2课：程序结构与基本输出
  • 程序的基本流程：输入、处理、输出。
  • Python代码格式：缩进的重要性, 注释 (#, ''' ''' 或 """ """), 字符串用引号。
  • print() 函数：输出文字、数字、表达式结果；默认换行；end 参数控制结尾。
  • type() 函数查看数据类型。
第3课：变量与数值型
  • 变量：存储数据的"小盒子"；命名规则（字母数字下划线，不能数字开头，区分大小写，不能用关键字）。
  • 赋值运算符 =。变量先赋值后使用。
  • 数值类型：整数 (int)，浮点数 (float)。
  • Python保留字（关键字）。
第4课：字符串
  • 字符串 (str)：用单引号或双引号创建。
  • 字符串操作：连接 (+)，重复 (*)。
  • 不同类型不能直接用 + 连接（如字符串和数字）。
  • 三引号创建多行字符串。
第5课：输入与类型转换
  • input() 函数获取用户输入（总是返回字符串）。
  • 类型转换：int(), float(), str()。
  • int() 转换特性：截断小数，不能转换含小数点的数字字符串。
第6课：数学运算
  • 算术运算符：+, -, *, / (结果为浮点数), // (整除), % (取余), ** (幂)。
  • 运算优先级（括号 > 幂 > 乘除余整除 > 加减）。
  • 复合赋值运算符（+=, *= 等）。
第7课：比较运算符
  • 布尔值：True, False。
  • 比较运算符：==, !=, >, <, >=, <= (结果为布尔值)。
  • True 相当于 1, False 相当于 0 在数值比较中。
第8课：逻辑运算符
  • 逻辑运算符：and, or, not。
  • 运算规则和真值表。
  • 短路求值和返回值特性 (例如 0 or 10 输出 10, True and 5 输出 5)。
  • 优先级：算术 > 比较 > not > and > or。
第9课：Turtle图形库
  • import turtle。
  • 移动：forward(), backward(), left(), right()。
  • 笔控：penup(), pendown()。
  • 定位：goto(), home()。
  • 绘图：circle(), dot()。
  • 外观：pensize(), pencolor(), fillcolor(), color()。
  • 填充：begin_fill(), end_fill()。
  • 其他：hideturtle(), speed(), done(), setup(), clear(), write()。
  • 坐标系概念。

快速问答/小测验（口头或书面）：

  • print(10 // 3) 的结果是什么？（ 3 ）

二、 小侦探修Bug：IDLE调试与程序分析

知识讲解：

同学们，写程序的时候，就像我们做手工或者写作文一样，有时候会不小心出点小差错，这些小差错在编程里叫做 Bug（虫子）。找到并修正这些Bug的过程，就叫做 调试 (Debugging)。

IDLE 环境下的错误提示：

  • 当我们的Python代码有比较明显的语法错误时，比如写错了单词、括号不配对、缩进不对等等，IDLE 在我们尝试运行程序之前可能就会用红色突出显示有问题的地方。
  • 如果程序在运行时出现问题，IDLE的交互模式窗口（Shell窗口）通常会显示错误信息 (Error Message)。这些信息看起来可能有点吓人，但它们其实是电脑在告诉我们哪里出错了，以及可能是什么原因。

常见的错误类型（一级会遇到的）：

  • SyntaxError (语法错误)：通常是代码不符合Python的写法规则。
    • 比如：primt("你好") (把 print 写成了 primt)
    • 比如：print("你好" (括号少了一边)
    • 比如：不正确的缩进 (IndentationError，它是 SyntaxError 的一种特殊情况)。
    • 使用非法标识符作为变量名也会导致 SyntaxError。
  • NameError (名称错误)：通常是因为使用了一个还没有被定义（赋值）的变量或函数名。
    • 比如：print(my_age)，但之前没有写过 my_age = 10 这样的赋值语句。
  • TypeError (类型错误)：通常是因为对不同类型的数据进行了不合适的操作。
    • 比如：print("年龄：" + 10) (尝试把字符串和整数用 + 连接)。
    • 比如：len(123) (尝试获取数字的长度，len() 通常用于序列如字符串、列表)。
  • ValueError (值错误)：通常是函数接收到的参数类型正确，但值不合适。
    • 比如：int("abc") (尝试把不能表示数字的字符串转换为整数)。

如何阅读错误信息？

  • 错误信息通常会告诉我们错误发生在哪个文件的哪一行。
  • 它还会指出错误的类型（比如 SyntaxError, NameError）。
  • 有时还会有一句简短的描述说明可能的原因。

简单的调试方法：使用 print() 语句

  • 这是初学者最常用也最直接的调试方法。
  • 如果你不确定程序执行到某一步时，某个变量的值到底是多少，或者程序有没有执行到某一段，你可以在代码的关键位置加入 print() 语句来输出这些信息。
  • 例如，如果一个计算结果不对，你可以在计算的每一步都 print() 一下中间变量的值，看看是哪一步算错了。 
    a = input("输入数字1: ")
b = input("输入数字2: ")
print("输入a后，a是：", a, "类型是：", type(a)) # 调试信息
print("输入b后，b是：", b, "类型是：", type(b)) # 调试信息
num_a = int(a)
num_b = int(b)
print("转换后，num_a是：", num_a, "num_b是：", num_b) # 调试信息
result = num_a + num_b
print("最终结果是：", result)
  • 调试完成后，记得把这些临时的 print() 语句删除或注释掉。

练习题1 (找出错误并改正)：

1. 下面的代码想计算面积，但有错误，请指出并改正：

length = input("请输入长度：")
width = "10"
area = length * width # 预期：如果输入长度为5，面积应为50
print(area)
错误原因：

length 是通过 input() 获取的字符串，width 也是字符串 "10"。字符串之间用 * 是重复操作，而不是数学乘法。需要将它们都转换为数值类型（如整数或浮点数）才能进行乘法运算。

改正后：
length_str = input("请输入长度：")
length = int(length_str) # 或 float(length_str)
width = 10 # 直接用数字10
area = length * width
print(area)

2. 下面的代码想打招呼，但有错误：

name = "小华"
age = 9
print("你好，" + name + "你今年" + age + "岁了。")
错误原因：

age 是整数类型，不能直接用 + 与字符串连接。需要用 str() 将其转换为字符串。

改正后：
name = "小华"
age = 9
print("你好，" + name + "，你今年" + str(age) + "岁了。") # 注意逗号和str()的使用

或者使用 print 的多参数特性：

name = "小华"
age = 9
print("你好，", name, "，你今年", age, "岁了。")

三、 解决问题的"小套路"

知识讲解：

当我们遇到一个编程问题时，特别是像编程题那样的要求，不要急着一上来就写代码。可以试试下面的步骤：

  1. 仔细阅读题目，理解要求：明确程序需要做什么？需要哪些输入？最终要输出什么？有什么特殊条件？
  2. 分解问题：把一个大问题拆分成几个小步骤。比如"计算找零钱"可以分解成：获取总金额和商品价格 -> 计算差额 -> 计算各种面额的数量。
  3. 思考每个小步骤需要用到的Python知识：这一步需要输入吗？用 input()。需要计算吗？用什么运算符？需要判断吗？用什么比较和逻辑？需要画图吗？用Turtle的什么命令？
  4. 先写出主要逻辑的"骨架"：把大的步骤用注释或者简单的代码搭起来。
  5. 逐步填充细节代码：一步一步实现每个小功能。
  6. 测试和调试：写一部分就运行一下，看看有没有错误，结果对不对。利用我们刚才学的调试方法。

实际问题举例：鸡妈妈数孩子问题

  • 题目："从前向后数到鸡妈妈自己是7，从后往前数，数到鸡妈妈自己是9。鸡妈妈到底有几个孩子？"
  • 分解：
    • 从前数，鸡妈妈是第7，说明她前面有 7-1=6 只。
    • 从后数，鸡妈妈是第9，说明她后面有 9-1=8 只。
    • 队伍总数 = 前面的 + 鸡妈妈自己 + 后面的 = 6 + 1 + 8 = 15 只。
    • 孩子数量 = 队伍总数 - 鸡妈妈自己 = 15 - 1 = 14 只。
  • Python代码实现（如果需要的话）： 
    front_pos = 7
back_pos = 9

front_count = front_pos - 1
back_count = back_pos - 1

total_in_line = front_count + 1 + back_count # 1是鸡妈妈自己
num_chicks = total_in_line - 1

print("鸡妈妈有", num_chicks, "个孩子。")

练习题2 (思考与分解)：

题目：小明想买一个玩具，玩具价格是50元。他有n元钱（由用户输入）。请判断小明的钱够不够买玩具，如果够，输出"可以买！"，如果不够，输出"钱不够哦。"

分解步骤（学生思考）：
  1. 需要什么输入？（小明有多少钱 n）
  2. 已知什么信息？（玩具价格 toy_price = 50）
  3. 怎么判断钱够不够？（比较 n 和 toy_price）
  4. 根据判断结果输出不同的话。

练习题3 (编程挑战 - 可选，或作为模拟题一部分)：

题目：绘制一个彩色的同心圆。要求：

  • 使用turtle库。
  • 绘制三个圆，它们的圆心都在画布中心(0,0)。
  • 第一个圆半径50，红色填充。
  • 第二个圆半径100，绿色填充，但它应该画在第一个圆的下面（先画大圆再画小圆，或者用抬笔定位）。
  • 第三个圆半径150，蓝色填充，画在第二个圆的下面。
  • 画笔粗细为3。
  • 完成后隐藏海龟。

提示：如何让后画的圆不覆盖先画的圆？可以考虑绘制的顺序，或者每次画圆前抬笔到圆的最低点（例如 goto(0, -radius)），然后画圆。

参考答案：
import turtle

turtle.speed(0) # 最快速度
turtle.hideturtle()
turtle.pensize(3)

# 第三个圆 (最底层)
turtle.penup()
turtle.goto(0, -150) # 定位到圆的底部开始点
turtle.pendown()
turtle.color("blue", "blue") # 画笔和填充都为蓝色
turtle.begin_fill()
turtle.circle(150)
turtle.end_fill()

# 第二个圆
turtle.penup()
turtle.goto(0, -100)
turtle.pendown()
turtle.color("green", "green") # 画笔和填充都为绿色
turtle.begin_fill()
turtle.circle(100)
turtle.end_fill()

# 第一个圆 (最顶层)
turtle.penup()
turtle.goto(0, -50)
turtle.pendown()
turtle.color("red", "red") # 画笔和填充都为红色
turtle.begin_fill()
turtle.circle(50)
turtle.end_fill()

turtle.done()
